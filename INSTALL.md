# Установка и запуск сервиса синхронизированных заметок

## Требования

- Python 3.7 или выше (поддержка Python 3.14 обеспечена)
- pip (менеджер пакетов Python)

## Установка

### 1. Клонирование репозитория (или загрузка файлов)

```bash
# Если используете Git
git clone <repository-url>
cd <repository-name>
```

### 2. Установка серверной части

```bash
cd server
pip install -r requirements.txt
```

### 3. Запуск сервера

```bash
cd server
python app.py
```

Сервер запустится на `http://localhost:5000`

### 4. Тестирование с консольным клиентом

```bash
cd client
pip install requests
python console_client.py
```

## Структура проекта

```
/workspace/
├── server/                 # Серверная часть
│   ├── app.py             # Основное приложение Flask
│   ├── requirements.txt   # Зависимости сервера
│   └── API.md             # Документация API
├── client/                # Клиентская часть
│   └── console_client.py  # Консольный клиент для тестирования
├── common/                # Общие файлы (по необходимости)
├── README.md              # Общее описание проекта
├── ARCHITECTURE.md        # Архитектурные решения
├── INSTALL.md             # Этот файл
└── API.md                 # API документация (альтернативное расположение)
```

## Использование консольного клиента

После запуска `python console_client.py` вы увидите меню:

1. **Регистрация** - создание нового аккаунта
2. **Вход** - аутентификация с существующим аккаунтом
3. **Создать заметку** - добавление новой заметки в формате Markdown
4. **Просмотреть все заметки** - список всех ваших заметок
5. **Просмотреть конкретную заметку** - детали одной заметки
6. **Обновить заметку** - редактирование существующей заметки
7. **Удалить заметку** - удаление заметки
8. **Выход** - завершение работы клиента

## Создание десктопного приложения (.deb для Linux)

Для создания .deb пакета для Linux:

1. Установите зависимости:
```bash
sudo apt-get install python3-dev python3-pip
pip3 install pyinstaller
```

2. Создайте исполняемый файл:
```bash
cd client
pyinstaller --onefile console_client.py
```

3. Создайте .deb пакет с помощью fpm или подобных инструментов

## Создание Android приложения

Для Android приложения рекомендуется использовать:

1. **Kivy** с Python для кроссплатформенной разработки
2. **BeeWare** для нативных приложений
3. **Chaquopy** для интеграции Python в Android Studio проект

Альтернативно, можно создать веб-приложение и обернуть его в WebView.

## Создание Windows приложения

Для Windows приложения:

1. Использовать PyInstaller для создания исполняемого файла:
```bash
pip install pyinstaller
pyinstaller --onefile --windowed console_client.py
```

2. Или создать полноценное GUI приложение с использованием PyQt/PySide

## Синхронизация между устройствами

Сервис обеспечивает синхронизацию через централизованный сервер:

1. При регистрации/входе пользователь получает уникальный ID
2. Все заметки привязаны к пользовательскому ID
3. При создании/изменении/удалении заметки изменения отправляются на сервер
4. При открытии приложения на другом устройстве происходит синхронизация с сервером

## Формат заметок

Заметки хранятся в формате Markdown:
- Поддержка заголовков: `# Заголовок`, `## Подзаголовок`
- Поддержка выделения: `*курсив*`, `**жирный**`
- Поддержка списков: `- элемент списка`
- Поддержка кода: ```код```, `inline код`
- Поддержка ссылок: `[текст](ссылка)`

## Безопасность

- Пароли хранятся в захешированном виде (используется werkzeug.security)
- Все заметки привязаны к конкретному пользователю
- Проверка авторизации при каждом запросе к заметкам

## Возможные проблемы и решения

1. **Сервер не запускается**: проверьте, установлены ли зависимости из requirements.txt
2. **Клиент не может подключиться к серверу**: убедитесь, что сервер запущен и доступен по адресу http://localhost:5000
3. **Ошибки доступа**: убедитесь, что используете правильный user_id при обращении к API
