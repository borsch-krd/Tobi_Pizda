# Архитектура сервиса синхронизированных заметок

## Общая архитектура

Сервис состоит из следующих компонентов:

1. **Серверная часть** - REST API на Python (Flask) с базой данных SQLite
2. **Клиентские приложения** - для Android, Windows и Linux
3. **Механизм синхронизации** - через REST API
4. **Формат хранения** - Markdown (.md)

## Серверная часть

Сервер реализует следующие функции:
- Аутентификация и авторизация пользователей
- CRUD операции с заметками
- Хранение заметок в формате Markdown
- Синхронизация между устройствами через API

## Клиентские приложения

### Android приложение
Для Android приложения рекомендуется использовать:
- Kotlin/Java с использованием Retrofit или Volley для HTTP запросов
- Room database для локального кэширования
- RecyclerView для отображения списка заметок
- Markdown рендеринг с помощью библиотеки

### Windows/Linux приложение
Для десктопных приложений можно использовать:
- Electron (JavaScript/TypeScript) для кроссплатформенности
- Qt с Python (PyQt/PySide)
- Или нативные решения (C# для Windows, C/GTK для Linux)

### Синхронизация
Синхронизация реализуется через REST API:
- При запуске приложения синхронизируем с сервером
- При изменении заметки отправляем обновление на сервер
- Периодическая синхронизация в фоне (опционально)

## Формат заметок

Заметки хранятся в формате Markdown:
- Текст может содержать Markdown разметку
- Сервер может конвертировать в HTML для отображения
- Поддержка заголовков, списков, кода, изображений и т.д.

## Установка и запуск

### Сервер
```bash
cd server
pip install -r requirements.txt
python app.py
```

### Консольный клиент (для тестирования)
```bash
cd client
pip install requests
python console_client.py
```
