# Полнофункциональный сервис синхронизированных заметок

## Обзор проекта

Этот проект представляет собой полноценный сервис для хранения и синхронизации заметок в формате Markdown между различными устройствами. Сервис включает в себя:

- **Серверную часть** на Python с REST API
- **Механизм аутентификации и авторизации**
- **Хранение заметок в формате Markdown**
- **Синхронизацию между устройствами**
- **Поддержку клиентских приложений** для Android, Windows и Linux

## Архитектура

### Серверная часть (`/server`)
- REST API на Flask
- База данных SQLite
- Аутентификация пользователей
- CRUD операции с заметками
- Поддержка Markdown формата

### Клиентская часть (`/client`)
- Консольный клиент для тестирования
- API клиент для интеграции в приложения

### Документация
- `API.md` - полная документация по API
- `ARCHITECTURE.md` - архитектурные решения
- `INSTALL.md` - инструкция по установке и запуску
- `MOBILE_DESKTOP_APPS.md` - руководство по созданию приложений

## Особенности

### 1. Формат заметок
- Заметки хранятся в формате Markdown
- Поддержка всех элементов Markdown: заголовки, списки, код, выделение текста и т.д.
- Возможность рендеринга в HTML

### 2. Синхронизация
- Централизованное хранение на сервере
- Уникальные ID пользователей
- Возможность доступа с разных устройств
- Защита данных через аутентификацию

### 3. Безопасность
- Хеширование паролей
- Проверка авторизации при каждом запросе
- Разграничение доступа к заметкам

### 4. Кроссплатформенность
- API позволяет создавать приложения для любых платформ
- Примеры реализации для Android, Windows и Linux
- Возможность создания веб-интерфейса

## Технологии

### Сервер
- **Python 3.7+** (с поддержкой Python 3.14)
- **Flask** - веб-фреймворк
- **SQLite** - база данных
- **Markdown** - обработка формата разметки

### Клиенты
- **requests** - для HTTP запросов
- **PyQt/PySide** - для десктопных GUI приложений
- **Kivy** - для мобильных приложений
- **BeeWare** - для нативных приложений

## Установка и запуск

1. **Установка зависимостей сервера:**
   ```bash
   cd server
   pip install -r requirements.txt
   ```

2. **Запуск сервера:**
   ```bash
   python app.py
   ```

3. **Тестирование с консольным клиентом:**
   ```bash
   cd client
   python console_client.py
   ```

## Возможности API

- Регистрация и аутентификация пользователей
- Создание, чтение, обновление и удаление заметок
- Получение заметок в формате Markdown и HTML
- Защита доступа к заметкам

## Создание приложений для разных платформ

### Android
- Использование Kivy, BeeWare или Chaquopy
- Интеграция с API через HTTP запросы
- Локальное хранилище для офлайн режима

### Windows
- PyQt/PySide для нативного интерфейса
- Упаковка в исполняемый файл с помощью PyInstaller

### Linux (.deb)
- Создание .deb пакета для установки
- Интеграция с системой рабочего стола

## Потенциальные доработки

1. **Шифрование данных** - для дополнительной безопасности
2. **Офлайн режим** - с последующей синхронизацией
3. **Теги и категории** - для организации заметок
4. **Совместное редактирование** - для работы в команде
5. **Веб-интерфейс** - для доступа через браузер
6. **Уведомления** - о изменениях и синхронизации

## Заключение

Этот проект предоставляет полную архитектуру для сервиса синхронизированных заметок с поддержкой Markdown формата. Серверная часть на Python обеспечивает надежное хранение и синхронизацию данных, а гибкий API позволяет создавать клиентские приложения для любой платформы.

Сервис полностью готов к использованию и может быть легко расширен дополнительными функциями в зависимости от требований.
